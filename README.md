# 📸 ScreenshotManager

**ScreenshotManager** — это умная и быстрая утилита для Windows, которая автоматизирует процесс сохранения скриншотов с использованием искусственного интеллекта (ИИ) для автоматического генерации описательных имен файлов.

Больше никаких бессмысленных имен вроде `Screenshot_1.png` или `Безымянный.png`. Программа анализирует содержимое скриншота или активное окно и дает файлу осмысленное имя!

---

## ✨ Основные возможности

*   **⚡ Моментальный захват**: Сохранение скриншотов из буфера обмена (автоперехват) или по горячей клавише.
*   **🧠 Умное именование файлов (ИИ)**: Использует локальную нейросеть **BLIP (Image Captioning)** через среду выполнения ONNX Runtime для автоматического описания происходящего на картинке (например, `a_cat_sitting_on_a_couch.png`).
*   **🖥️ Именование по активному окну**: Автоматически определяет имя приложения и заголовок окна, в котором был сделан скриншот (например, `chrome_github_screenshotmanager.png`).
*   **📅 Классическое именование**: Шаблоны с датой и временем.
*   **🛠️ Гибкие настройки**: Настройка папки сохранения, выбор горячих клавиш, включение/выключение автозапуска.
*   **🌐 Многоязычность**: Полная поддержка русского и английского языков.

## 🚀 Инструкция по установке

### Вариант 1: Установка через инсталлятор (Рекомендуется)
1. Перейдите в раздел **[Releases](https://github.com/rmshkmx/ScreenshotManager/releases)** на GitHub.
2. Скачайте файл `ScreenshotManager_Setup.exe`.
3. Запустите установщик и следуйте инструкциям на экране. Вы можете выбрать создание ярлыка на рабочем столе и автозапуск программы при старте Windows.

### Вариант 2: Портативная версия (Без установки)
1. Скачайте архив `ScreenshotManager_Portable.zip` из раздела **Releases**.
2. Распакуйте архив в любую удобную папку.
3. Запустите файл `ScreenshotManager.exe`.

## 🤖 Руководство по настройке и использованию ИИ

Для того чтобы приложение могло распознавать содержимое ваших скриншотов, используется модель **BLIP**. Сборка оптимизирована для работы локально на вашем процессоре (CPU) через ONNX Runtime, так что данные не отправляются на сторонние сервера.

### Шаг 1. Включение режима ИИ в приложении
1. Откройте настройки приложения **ScreenshotManager** (иконка в трее или через главное окно).
2. Перейдите в раздел настроек именования.
3. Выберите режим именования **AI (ИИ)** или **Combined (Комбинированный)**.

### Шаг 2. Скачивание модели
После выбора ИИ-режима приложение предложит загрузить файлы модели.
*   **Автоматическое скачивание**: Приложение само скачает необходимые файлы с Hugging Face (~650 МБ суммарно). Вы увидите прогресс-бар загрузки.
*   **Где хранятся файлы**: Модель сохраняется локально в вашей папке пользователя:
    `C:\Users\<Имя_Пользователя>\AppData\Local\ScreenshotManager\Models\blip-captioning-base`

> [!NOTE]
> Если загрузка не начинается или прерывается, вы можете сгенерировать персональный токен доступа на сайте Hugging Face и вставить его в поле **Hugging Face Token** в настройках программы. Это решит проблему с ограничениями на скачивание со стороны Hugging Face.

## 🛠️ Руководство для разработчика (Как делать коммиты и релизы)

Если ты хочешь вносить изменения в проект самостоятельно, вот краткий гайд по работе с Git в этом репозитории.

### Как делать коммиты изменений

Открой терминал (например, PowerShell или Git Bash) в папке проекта:

1. **Проверить статус изменений**:
   ```bash
   git status
   ```
2. **Добавить измененные файлы к коммиту**:
   * Чтобы добавить все измененные файлы:
     ```bash
     git add .
     ```
   * Или добавь конкретный файл:
     ```bash
     git add Path/To/File.cs
     ```
3. **Создать коммит** (сохранить изменения локально с описанием):
   ```bash
   git commit -m "Описание твоих изменений (например: исправил баг в настройках)"
   ```
4. **Отправить изменения на GitHub**:
   ```bash
   git push origin main
   ```

---

### Как выпустить новый релиз (Автоматическая сборка установщика)

Благодаря настроенному **GitHub Actions**, тебе не нужно собирать установщик вручную. Сервер GitHub сделает это за тебя при создании нового тега версии:

1. Убедись, что все изменения закоммичены и отправлены в ветку `main`.
2. Создай тег версии (например, `v1.0.0` для первой версии, `v1.0.1` для патча):
   ```bash
   git tag v1.0.0
   ```
3. Отправь созданный тег на GitHub:
   ```bash
   git push origin v1.0.0
   ```

После этого:
- Перейди во вкладку **Actions** в твоем репозитории на GitHub. Ты увидишь запущенный процесс сборки `Release Setup`.
- Скрипт соберет проект, создаст портативный архив и установщик `ScreenshotManager_Setup.exe` с помощью Inno Setup.
- По завершении (около 2-3 минут) во вкладке **Releases** автоматически появится опубликованная версия программы с готовыми к скачиванию файлами.
